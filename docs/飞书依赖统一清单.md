# 飞书依赖统一清单

更新时间：2026-04-25 23:59

## 使用规则

以后只要系统提示“缺东西”，统一按这个格式写：

```text
缺什么 -> 从哪里来 -> 谁能提供 -> 怎么验证 -> 卡住哪个能力
```

飞书依赖通常来自四个地方：

- 飞书开放平台应用后台：App 凭证、API 权限、机器人能力、事件订阅。
- 真实业务对象：群聊、文档、多维表格、会议妙记、任务、日历。
- 对象 URL 或详情页：从链接里解析 `chat_id`、`doc_id`、`app_token`、`table_id`、`minute_token`。
- 用户授权：会议妙记、个人日历、个人任务等经常需要用户身份，而不只是机器人身份。

## 9. 最小提供清单

为了继续从“已连接”推进到“真实业务验收”，当前最少需要：

1. 真实群回复确认：在测试群 `oc_17a66849f49461ef2b6f42f89e0f48f4` 再 @ 小龙虾一次。
2. 一个测试文档 URL 或 `doc_id`，并授权给机器人或应用。
3. 一个会议妙记 URL 或 `minute_token`。
4. 一个测试任务 GUID。
5. 一个日历 ID，或完成用户身份授权。
6. 在飞书开放平台确认并发布缺失权限：`im:*`、`minutes:*`、`task:*`、`calendar:*` 中本清单标注为缺的项。
