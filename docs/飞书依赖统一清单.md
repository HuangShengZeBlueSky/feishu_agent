# 飞书依赖统一清单

更新时间：2026-04-25

## 使用规则

以后只要系统提示“缺东西”，必须按这张表的格式写：

```text
缺什么 -> 从哪里来 -> 谁能提供 -> 怎么验证 -> 卡住哪个能力
```

不要只写“缺 token / 缺权限 / 缺 ID”。飞书依赖通常来自四个地方：

1. 飞书开放平台应用后台：App ID、App Secret、API 权限、机器人能力、事件订阅。
2. 飞书真实业务对象：群聊、文档、多维表格、会议妙记、任务、日历。
3. 飞书对象 URL 或对象详情页：从链接里解析 `chat_id`、`doc_id`、`app_token`、`table_id`、`minute_token`。
4. 用户授权：会议妙记、个人日历、个人任务等可能需要用户身份，而不是只有机器人身份。

## 1. 基础凭证

| 依赖 | 中文含义 | 从哪里来 | 谁能提供 | 当前状态 | 怎么验证 |
|---|---|---|---|---:|---|
| `FEISHU_APP_ID` | 飞书应用 ID | 飞书开放平台 -> 目标应用 -> 凭证与基础信息；本项目模板位于 `resources/config/飞书环境变量模板.env` | 应用管理员/开发者 | 已存在，当前 App ID 已写入模板 | `tools/feishu_capability_probe.py` 能换取 token |
| `FEISHU_APP_SECRET` | 飞书应用密钥 | 飞书开放平台 -> 目标应用 -> 凭证与基础信息；本机真实值位于被忽略的 `resources/config/本地飞书密钥.env` | 应用管理员/开发者 | 已存在于本机忽略文件 | 不打印密钥，只验证 token |
| `tenant_access_token` | 租户访问令牌 | 用 App ID + App Secret 调飞书鉴权接口自动换取 | 脚本自动生成 | 可自动获取 | `auth.tenant_access_token status=ok` |
| OpenClaw Gateway | OpenClaw 本地服务 | 本机 OpenClaw 安装与 Gateway 服务 | 本机环境 | 未运行 | `openclaw gateway status` |
| OpenClaw Feishu plugin | OpenClaw 飞书通道插件 | OpenClaw bundled plugin 或插件安装 | 本机环境 | Windows 原生加载失败 | `channels status` / `channels capabilities` |

## 2. 多维表格

| 依赖 | 中文含义 | 从哪里来 | 谁能提供 | 当前状态 | 卡住什么 |
|---|---|---|---|---:|---|
| `FEISHU_BITABLE_APP_TOKEN` | 多维表格 app token | 多维表格 URL，通常在 `base/`、`bitable/` 链接中；也可由 API 创建后返回 | 表格创建者/项目管理员 | 缺 | 真实写入无法定位目标多维表格 |
| `FEISHU_ACTION_TABLE_ID` | 行动项表 ID | 多维表格内具体数据表 URL 或 API 查询 tables 返回 | 表格创建者/项目管理员 | 缺 | `Project_Action_Items` 无法写入 |
| `FEISHU_DECISION_TABLE_ID` | 决策表 ID | 多维表格内具体数据表 URL 或 API 查询 tables 返回 | 表格创建者/项目管理员 | 缺 | `Project_Decisions` 无法写入 |
| `FEISHU_RISK_TABLE_ID` | 风险表 ID | 多维表格内具体数据表 URL 或 API 查询 tables 返回 | 表格创建者/项目管理员 | 缺 | `Project_Risks` 无法写入 |
| 多维表格写权限 | 创建表/字段/记录/更新记录 | 飞书开放平台权限 + 多维表格对象授权 | 应用管理员 + 表格所有者 | 已实测可创建 app/table/record/update | 真实写入 |

验证命令：

```powershell
$env:FEISHU_ENABLE_WRITES="1"
python tools\feishu_capability_probe.py --json
```

## 3. 群聊与卡片

| 依赖 | 中文含义 | 从哪里来 | 谁能提供 | 当前状态 | 卡住什么 |
|---|---|---|---|---:|---|
| `FEISHU_TEST_CHAT_ID` / `FEISHU_CHAT_ID` | 目标项目群 ID | 群信息、机器人被拉入群后的事件、目录/消息 API、或飞书后台工具 | 群主/项目管理员/开发者 | 缺 | 不能发会后摘要、风险提醒、确认卡片 |
| 机器人加入目标群 | 让机器人具备群内可见性 | 飞书客户端把机器人加入群，或开放平台配置机器人能力 | 群主/应用管理员 | 缺确认 | 机器人不在群里通常无法发群消息 |
| `im:message:send_as_bot` | 机器人发消息权限 | 飞书开放平台 -> 权限管理 | 应用管理员 | 待确认 | 文本消息/卡片推送 |
| `im:message` | 消息基础权限 | 飞书开放平台 -> 权限管理 | 应用管理员 | 待确认 | 消息读写基础能力 |
| `im:message.group_at_msg:readonly` | 读取群里 @ 机器人消息 | 飞书开放平台 -> 权限管理 | 应用管理员 | 后续需要 | 群指令、手动触发 |
| `im:resource` | 上传图片/附件资源 | 飞书开放平台 -> 权限管理 | 应用管理员 | 后续可选 | 带附件/图片的卡片 |

安全开关：

```powershell
$env:FEISHU_ENABLE_WRITES="1"
$env:FEISHU_ENABLE_CARD_PUSH="1"
```

## 4. 文档

| 依赖 | 中文含义 | 从哪里来 | 谁能提供 | 当前状态 | 卡住什么 |
|---|---|---|---|---:|---|
| `FEISHU_TEST_DOC_ID` / `FEISHU_DOC_ID` | 文档 ID | 飞书文档 URL，可从 `docx/` 链接解析 | 文档所有者/项目管理员 | 缺 | 不能读取项目背景文档 |
| 文档授权给机器人/应用 | 让应用能访问文档 | 文档分享设置、文档所在空间授权、应用权限 | 文档所有者/空间管理员 | 缺确认 | tenant token 可能拿不到文档 |
| 新版文档读取权限 | 读取 docx 原文或 block | 飞书开放平台 -> 权限管理 | 应用管理员 | 待确认 | `docx.raw_content` / block 读取 |

第一版规则：只读 `Project_Context_Sources` 登记过的文档，不默认搜全公司文档，不自动改文档。

## 5. 会议妙记

| 依赖 | 中文含义 | 从哪里来 | 谁能提供 | 当前状态 | 卡住什么 |
|---|---|---|---|---:|---|
| `FEISHU_TEST_MINUTE_TOKEN` / `FEISHU_MEETING_MINUTE_TOKEN` | 会议妙记 token | 会议妙记 URL，一般可从链接中解析 `minute_token` | 会议组织者/妙记可见用户 | 缺 | 不能读取真实会议内容 |
| `minutes:minutes.basic:read` | 读取妙记基础信息/摘要类信息 | 飞书开放平台 -> 权限管理 | 应用管理员 | 当前缺 | 妙记标题、基础信息、摘要 |
| `minutes:minutes:readonly` / `minutes:minutes` | 读取妙记内容 | 飞书开放平台 -> 权限管理 | 应用管理员 | 当前缺 | 章节、内容、行动项等 |
| `minutes:minutes.transcript:export` | 导出逐字稿 | 飞书开放平台 -> 权限管理 | 应用管理员 | 当前缺 | 逐字稿读取 |
| `minutes:minute:download` | 下载妙记相关内容 | 飞书开放平台 -> 权限管理 | 应用管理员 | 当前缺 | 下载型内容接口 |
| 用户身份授权 | 用户可见妙记访问权 | 飞书 OAuth 用户授权 | 妙记可见用户/应用管理员 | 很可能需要 | 机器人 tenant token 不一定能看用户妙记 |

验证方式：先给一条真实妙记链接或 token，再运行：

```powershell
$env:FEISHU_TEST_MINUTE_TOKEN="..."
python tools\feishu_capability_probe.py --json
```

## 6. 任务

| 依赖 | 中文含义 | 从哪里来 | 谁能提供 | 当前状态 | 卡住什么 |
|---|---|---|---|---:|---|
| `FEISHU_TEST_TASK_GUID` / `FEISHU_TASK_GUID` | 飞书任务 GUID | 任务 URL、任务详情、任务 API 返回 | 任务创建者/项目管理员 | 缺 | 不能读取/同步真实任务 |
| `task:task:read` | 读取任务权限 | 飞书开放平台 -> 权限管理 | 应用管理员 | 当前缺 | 读取任务 |
| `task:task:write` | 创建/更新任务权限 | 飞书开放平台 -> 权限管理 | 应用管理员 | 当前缺 | 创建行动项任务、更新任务状态 |

安全规则：只有 `confidence >= 0.8`、有 `owner_text` 或真实 `owner_user_id`、有 `due_date` 的行动项，才允许进入任务创建候选；真实创建任务前必须显式参数确认。

## 7. 日历

| 依赖 | 中文含义 | 从哪里来 | 谁能提供 | 当前状态 | 卡住什么 |
|---|---|---|---|---:|---|
| `FEISHU_TEST_CALENDAR_ID` | 日历 ID | 日历 URL、日历设置、日历 API 返回 | 日历所有者/项目管理员 | 缺 | 会前提醒、会前背景卡片 |
| 默认日历访问 | 读取当前用户/机器人可见日历 | 日历授权或用户授权 | 日历所有者/用户 | 缺 | 自动找会议 |
| `calendar:calendar:readonly` | 读取日历权限 | 飞书开放平台 -> 权限管理 | 应用管理员 | 当前缺 | 读取日历列表/事件 |
| `calendar:calendar:read` | 读取日历权限 | 飞书开放平台 -> 权限管理 | 应用管理员 | 当前缺 | 读取日历列表/事件 |
| `calendar:calendar.calendar:readonly` | 读取日历相关资源 | 飞书开放平台 -> 权限管理 | 应用管理员 | 当前缺 | 日历对象读取 |
| `calendar:calendar` | 日历读写总权限 | 飞书开放平台 -> 权限管理 | 应用管理员 | 可选 | 创建/修改日历事件 |

第一版策略：日历不是阶段 1/2 blocker。先用手动命令和本地定时任务，后续再做会前触发。

## 8. 最小提供清单

为了从本地 dry-run 推进到真实飞书 MVP，只需要先给这些：

1. 一个项目多维表格的 `app_token`。
2. 三张表的 `table_id`：行动项表、决策表、风险表。
3. 一个测试项目群 `chat_id`，并确认机器人已加入。
4. 一个会议妙记链接或 `minute_token`。
5. 在飞书开放平台补齐并发布这些权限：`minutes:*`、`task:*`、`calendar:*`、`im:*` 中本清单标注为当前缺的项。
