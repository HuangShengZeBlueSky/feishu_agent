## 仍缺

| 缺什么 | 从哪里来 | 谁能提供 | 怎么验证 | 卡住哪个能力 |
|---|---|---|---|---|
| 你真实创建的妙记 token | 飞书妙记链接，通常包含 `/minutes/` | 会议组织者或可见用户 | 能读取妙记标题、章节、摘要或逐字稿 | 用真实会议妙记替代 seed 会议文本 |
| 真实项目文档/表格/任务链接 | 你手动创建或授权的项目资料 | 你或项目成员 | 小龙虾能登记并在 Base 中看到真实 source | 让 A/B/D 输出更贴近真实项目 |
| 参会人 open_id / freebusy 授权 | 通讯录或用户授权 | 参会人/管理员 | `calendar +freebusy` 能查忙闲 | 自动约会生成真实候选时间 |
| 真实群回复确认 | 群 `1` 里 @ 飞书机器人 `会议` | 你 | 群里收到 `会议` 的自然语言回复 | 群内触发和卡片推送闭环 |

## 当前安全规则

- 不再从 `resources/mock` 读取业务数据。
- 缺真实数据时，只允许创建真实飞书 Base seed 数据，并明确标记 `feishu-seed://...`。
- 真实创建任务、真实创建文档、真实创建日程、真实发群消息必须能追溯到用户自然语言指令或显式命令。
- 没有真实日历忙闲时，不编造候选时间。
- 没有真实用户 ID 时，只写 `owner_text`，不编造 `owner_user_id`。
