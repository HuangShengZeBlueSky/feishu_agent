# 小龙虾提示词与技能路由

更新时间：2026-04-25 23:59

## 结论

可以实现。实现方式分成三层：

1. OpenClaw 飞书通道：接收飞书消息、控制群白名单、发送回复和卡片。
2. 小龙虾提示词：判断用户意图、选择工作流、先预览再写入。
3. 飞书 CLI 与 skills：访问飞书对象，包括多维表格、文档、群消息、妙记、任务、日历。

当前第 1 层和 mock 业务链路已经可验收。第 2 层需要把下面的提示词固化到 OpenClaw agent 启动上下文。第 3 层里 bot 身份已可用；用户身份 OAuth 还没登录，所以个人日历、个人妙记、个人任务读取先作为缺口保留。

## 建议系统提示词

```text
你是“小龙虾项目上下文管家”，只服务已授权的飞书项目群和项目上下文清单。

你的目标不是闲聊，而是维护团队项目上下文、行动项、风险和会议闭环。

工作规则：
1. 先读取项目上下文清单，不做全局乱搜。
2. 所有行动项、决策、风险都必须带 source_quote 或 source_url。
3. owner_text 可以用中文负责人名；owner_user_id 不允许编造。
4. 创建任务、创建文档、推送群卡片、修改多维表格前必须先输出预览。
5. 默认 dry-run；只有用户明确确认后才执行真实写操作。
6. 每次运行都要生成 run_log，记录输入、输出、写入计划、实际写入和错误。
7. 当缺少权限或对象 ID 时，按“缺什么 -> 从哪里来 -> 谁提供 -> 怎么验证 -> 卡住哪个能力”列出。
```

## A/B/D 实现路径

### 方向 A：周期性智能总结与洞察

目标：每周自动生成“本周工作总结与下周风险洞察”卡片。

MVP 路径：

1. 读取项目推进总表、决策表、风险表。
2. 汇总本周完成、新增、延期、阻塞、高风险、缺负责人、缺截止时间事项。
3. 生成周报卡片预览。
4. 用户确认后推送到测试群。
5. 后续接 Windows 定时任务或飞书事件。

当前命令：

```powershell
python -m src.cli risk-weekly-insight
```

### 方向 B：会议与项目全链路伴侣

目标：会前推背景，会后抽行动项并进入闭环。

MVP 路径：

1. 会前：基于会议标题和项目 ID，从项目上下文清单找相关文档、历史决策、未完成事项和风险。
2. 会后：读取本地会议文本或飞书会议纪要文本。
3. 抽取决策、行动项、风险、待确认问题。
4. 写入多维表格；创建飞书任务前先输出确认清单。

当前命令：

```powershell
python -m src.cli pre-meeting-card --meeting-title 支付项目评审会
python -m src.cli post-meeting-local --minutes-file resources\mock\sample_minutes.txt --write-bitable --dry-run
```

### 方向 D：团队待办中枢与进展自动对账

目标：维护一张可追踪、可分派、可预警的重点事项推进总表。

MVP 路径：

1. 从群聊、文档正文、文档评论、会议纪要、任务列表抽取 Todo、阻塞和风险。
2. 与项目推进总表做标题归一化和关键词匹配。
3. 输出新增、更新、合并、关闭、需要人工确认的 diff。
4. 用户确认后写入多维表格。

当前命令：

```powershell
python -m src.cli reconcile-project-table
```

## 飞书 skill 路由

| 能力 | 使用的 skill/CLI | 当前状态 |
|---|---|---|
| 多维表格读写 | `lark-base` 或当前 OpenAPI 适配器 | 已真实创建和写入 |
| 群消息/卡片 | `lark-im` 或 OpenClaw Feishu channel | OpenClaw 可收消息，测试群已加入白名单；真实回复待你再发消息验收 |
| 文档读取/创建 | `lark-doc`、`lark-drive` | 缺真实文档 URL/授权 |
| 妙记摘要/章节/待办/逐字稿 | `lark-minutes` + `lark-vc` | 缺真实 `minute_token` 和用户授权 |
| 任务创建/更新 | `lark-task` | 缺任务权限和真实任务对象 |
| 日历会前触发 | `lark-calendar` | 缺用户 OAuth 或可访问日历 |
| 事件触发 | OpenClaw WebSocket / `lark-event` | OpenClaw WebSocket 已运行；业务事件流后续接入 |
