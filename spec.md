3. 项目推进总表维护与自动对账

对应方向：D，部分 A

这是方向 D 的核心例子，必须加强。

功能

周期性从多个来源识别 Todo、阻塞和风险，并维护项目推进总表。

数据源包括：

群聊；
文档正文；
文档评论；
会议纪要；
飞书任务；
手工录入记录。

它不是简单新增记录，而是要做自动对账：

新事项新增；
已存在事项更新；
重复事项合并；
已完成事项关闭；
状态变化同步；
来源证据追加。
实现

执行流程：

读取项目上下文清单
→ 拉取 enabled=true 的来源
→ 抽取 Todo、阻塞、风险
→ 与项目推进总表现有记录比对
→ 判断新增、更新、合并、关闭
→ dry-run 输出 diff
→ 确认后写入多维表格

必须输出 diff：

新增：3 条
更新：4 条
合并：2 条
关闭：1 条
需要人工确认：2 条
测试

输入一段模拟群聊：

产品：验收材料这周要准备好，不然下周客户评审会受影响。

前端：我这边等错误码文档，没文档没法继续联调。
测试：联调环境今天又挂了，测试进度可能要延后。
产品：验收材料谁负责？目前好像没人接。

再输入一段模拟文档评论：

评论 2：验收材料缺少负责人，建议今天定下来。

再输入一组模拟任务：

    "status": "doing",

  },
  {
    "title": "确认验收材料负责人",
    "owner_text": "",
    "status": "todo",
    "due_date": ""
  }
]

验收标准：

检查项	通过标准
Todo 抽取	识别验收材料、错误码文档
阻塞识别	识别前端等待错误码文档、联调环境挂了
风险识别	识别客户评审受影响、测试延期

4. 周期性风险巡检与周报洞察

对应方向：A、D

这是对原“任务延期提醒”的升级，用来补齐方向 A。

功能

分成两个层次：

A. 每日风险巡检

每天检查：

延期事项；
缺负责人事项；
缺截止时间事项；
阻塞事项；
高风险事项；

每周汇总：

本周任务流转；
本周核心文档变化；
本周风险；
下周建议关注事项。
实现

每日流程：

读取项目推进总表
→ 筛选延期、阻塞、缺信息、高风险
→ 生成今日风险卡片
→ dry-run 或推送

每周流程：

读取本周项目推进总表变化
→ 读取本周会议决策
→ 读取本周文档更新摘要
→ 读取本周任务状态变化
→ 生成周报洞察卡片

5. 自动约会与角色忙闲模拟

对应方向：B

这个例子保留，但优先级最低。
因为你现在只有一个人，所以第一版必须支持“角色模拟模式”。

功能

根据角色忙闲表推荐会议时间，生成会议草稿。

角色包括：

产品；
后端；
前端；
测试；
项目经理。

第一版不要求真实创建会议。

实现

执行流程：

读取 simulated_calendar.json
→ 找到所有角色都有空的时间段
→ 推荐 1 到 3 个候选时间
→ 生成会议草稿
→ 关联项目背景链接
→ 不直接创建真实日程

如果未来有真实飞书用户，再切换到真实日历模式。

测试

四、给 AI / Codex 的最新执行清单

可以直接给 Codex 这段。

请按以下 6 个例子实现和验证“小龙虾项目上下文管家”。

公共要求：
1. 所有功能必须支持 mock 模式和 feishu 模式。
2. 所有真实写操作默认 dry-run。
3. 创建任务、创建文档、发送群消息、创建会议前必须先输出预览。
4. 每条行动项、决策、风险必须保留 source_quote 或 source_url。
5. 负责人字段必须同时支持 owner_text 和 owner_user_id。
6. 当前只有一个真实飞书用户，因此 mock 模式下使用产品、后端、前端、测试、项目经理这些角色模拟团队。
7. 不允许编造真实 user_id。
8. 每次运行必须输出 run_log，包括输入、输出、写入计划、实际写入结果、错误信息。

需要实现的 6 个例子：

1. 会后行动项闭环
- 输入：会议纪要文本或飞书会议纪要文档。
- 输出：决策、行动项、风险、待确认问题。
- 写入：项目推进总表、会议决策表、项目风险表。
- 验收：能识别行动项、负责人、截止时间、风险和来源证据。

2. 会前背景知识卡片
- 输入：会议标题、项目 ID、项目上下文清单。
- 输出：会前背景卡片。
- 内容：上次决策、未完成事项、当前风险、建议确认问题、来源链接。
- 验收：必须从项目上下文清单读取资料，不允许全局乱搜。

3. 项目推进总表维护与自动对账
- 输入：群聊文本、文档正文、文档评论、会议纪要、任务列表。
- 输出：新增、更新、合并、关闭的 diff。
- 写入：项目推进总表。
- 验收：支持去重、状态同步、来源证据、文档评论抽取。

4. 周期性风险巡检与周报洞察
- 输入：项目推进总表、会议决策表、项目风险表、文档更新摘要、任务状态变化。
- 输出：今日风险卡片和每周工作总结与下周风险洞察卡片。
- 验收：识别延期、阻塞、缺负责人、缺截止时间、高风险，并能生成周报。

5. 自动约会与角色忙闲模拟
- 输入：角色忙闲表 simulated_calendar.json。
- 输出：候选会议时间和会议草稿。
- 验收：当前只做模拟模式，不直接创建真实会议，不编造真实用户。

6. 文档生成、更新与知识沉淀
- 输入：会议纪要或周报数据。
- 输出：会后总结文档或周报文档。
- 写入：飞书文档，并将链接登记到项目上下文清单。
- 验收：创建前有预览，创建后返回链接，关键结论有来源。
